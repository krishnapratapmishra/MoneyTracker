11 — Colors to Avoid
Anti-Patterns
These feel dated, sterile, or cause visual discomfort in wealth/lifestyle apps. Hard rules for the team.
✗
Pure Gray #808080
Feels sterile, enterprise, lifeless. Replace with slate blue-grays that have personality.
✗
Pure Yellow #ffff00
Neon yellow feels cheap and is unreadable. Use amber #f59e0b instead.
✗
Pure Red #ff0000
Too aggressive for financial data. Use rose #f43f5e or #ef4444 for danger states.
✗
Pure Green #00ff00
Harsh, Matrix-like. Use emerald #10b981 — much richer and trustworthy.
✗
Flat White #f0f0f0
Zero visual hierarchy. Tint your backgrounds — use #f0f4ff or #f8faff instead.
✗
Rainbow Gradients
Using 4+ hue gradients on single elements. Looks like a startup from 2015. Max 2-hue gradients.
✗
Pure Black #000000
True black looks harsh on screens. Use #0b0e1a or #111827 — dark but with personality.
✗
Pure Blue #0000ff
Looks like a browser link. Use indigo #4f46e5 or sky #0ea5e9 — refined alternatives.

13 — Accessibility
Contrast & Readability Guide
All approved text/background pairs pass WCAG AA (4.5:1 for body text, 3:1 for large text). Test before overriding.
Text Color Background Ratio Rating Usage
#f0f4ff#0b0e1a18.7:1AAA ✓Body text on dark BG
#a5b4fc#1118277.2:1AAA ✓Indigo text on surface
#67e8f9#0c4a6e5.8:1AA ✓Cyan text on cyan card
#fcd34d#78350f6.1:1AA ✓Amber text on amber card
#d1fae5#064e3b9.4:1AAA ✓Emerald card content
#0f172a#e0e7ff14.2:1AAA ✓Light mode indigo card
#8896b3#0b0e1a4.6:1AA ✓Muted label on dark BG
Golden Rules
✓ Body text: min 4.5:1 contrast
✓ Large text (18px+): min 3:1 contrast
✓ Never use color alone to convey meaning
✓ Add icons + text to all state indicators
✓ Test every card color in both themes
✓ Use APCA contrast for modern apps
Color-Blind Safe Pairs
✓ Use indigo+amber (not red+green) for gain/loss
✓ Add ↑↓ arrows alongside color changes
✓ Use patterns in charts when possible
✓ Avoid red+green as the only differentiator
✓ Teal+Orange is the safe green+red alternative
✓ Test with Sim Daltonism or Chrome DevTools
